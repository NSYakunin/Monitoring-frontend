## Описание проекта
**Monitoring Frontend** — это одностраничное приложение (SPA), написанное на **React** и **TypeScript**. Оно обеспечивает удобный пользовательский интерфейс для взаимодействия с серверным **Monitoring Web API**, включая:
- Авторизацию, отображение списка работ, уведомлений и т.д.
- Гибкую фильтрацию и сортировку данных.
- Визуальные компоненты для просмотра и редактирования сущностей системы.
- Реализацию **чата** на базе **SignalR** (через соответствующий хаб на бэкенде).

---

## Основные особенности
- **React (TypeScript)**: Статическая типизация, повышенная надёжность кода.
- **SignalR**-чат: Обмен сообщениями в реальном времени (отдельный компонент/файл чата).
- **Динамические таблицы**: Drag-and-drop, выбор элементов для формирования отчётов.
- **Удобные фильтры и поиск**: По исполнителям, по текстовому вводу и другим параметрам.
- **Управление состоянием** (при необходимости с помощью Redux или контекста).

## Используемые технологии
| Технология          | Описание                                                  |
|---------------------|-----------------------------------------------------------|
| **React**           | JavaScript-библиотека для создания пользовательских интерфейсов. |
| **TypeScript**      | Надстройка над JavaScript, добавляющая статическую типизацию.    |
| **SignalR (client)**| Подключение к SignalR-хабу, реализованному на бэкенде.            |
| **HTML/CSS**        | Разметка и стилизация приложения.                                 |
| **Axios / Fetch**   | Для отправки HTTP-запросов к Web API (REST).                     |
| **React Router**    | (Опционально) Маршрутизация в одностраничном приложении.         |
